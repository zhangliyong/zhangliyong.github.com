Generalized Suffix Tree

Fri 05 July 2013
By zhang liyong

In algorithm.

后缀树可以用来快速的对字符串进行搜索，详细信息推荐阅读：http://en.wikipedia.org/wiki/Suffix_tree，

后缀树的搜索非常快，但是构造后缀树比较复杂， Ukkonen提出的构造后缀树的算法时间复杂度为O(nlogn), 构造原理推荐阅读 http://stackoverflow.com/questions/9452701/ukkonens-suffix-tree-algorithm-in-plain-english

本文提供了另一种构造generalized suffix tree的方法，此种方式不需要为每个字符串添加惟一的字符，所以可以为任意数量的字符串构造后缀树。 构造原理依然采用Ukkonen提出的。

在继续阅读前确保你已经理解后缀树的构造原理。

suffix tree只表示一个字符串， 在很多情况下我们需要为多个字符串构造后缀树 generalized suffix tree，例如查询包含'abc'的所有字符串， 当需要为n个字符串（如s1, s2, s3, ..., sn）构造时，目前通用的做法是找到n个不同的字符, u1, u2,..., un， 这n个字符不在这n个字符串中，然后组成一个字符串：s=s1u1s2u2.....snun，然后为s构造后缀树。

为"abc#abcde$"构造后缀树，如图:

当有少量字符串的时候，这种方式比较实用，我们可以很轻易找到n个不同的字符，但当有大量字符串时， 我们很难找到这么多不同的字符。目前unicode的码点是从0到0x10ffff，也就是unicode最多能表示1114111个字符，仅仅是百万级。

所以通过不同字符来连接字符串的方案对大量字符串并不实用。

为此我们提供了另一种解决方案。 我们需要为每个字符串设置一个惟一的索引，用于标识此字符串，建立起索引到字符串的映射。

以字符串"ab", "abc"为例, 设置"ab"的索引为1, "abc"的索引为2，为其创建后缀树为：

当时行搜索时，从根结点向下查找，当成功到达某一结点时，此结点及其所有子结点中的索引值即是查找结果。

例如查找'b'时，可找到索引1，2.

下面介绍构造此generalize suffix tree的过程。

给定字符串"aba"，其索引为0

  1. 根据Ukkonen算法构造后缀树。
  2. 将索引加入后缀树中的所有叶结点。
    • 如果最后一个字符加入后，remainder为0，则不需要做任何处理，此时后缀树已经创建成功；
  4. 如果remainder不为0, 则还有remainder个后缀没有对应到索引，根据需要创建中间结点，并将索引加到中间结点中。

构造完后缀树，并将索引加到叶结点后，如图：

此时remainder为1，active point为(root, aba, 1)，需要创建中间结点，并将索引加入其中，如图：

如果某一suffix结束于某一结点，即此时active_length为0，则只需要在此结点加入索引即可，不需要创建；上例中active_length为1，则需要创建一中间结点。

下面给出程序的伪代码：

for char in text:
    remainder ++
    while remainder > 0:
        canonize // make sure the active_length is no longer than active_edge

        if active_length is 0
            edge = get edge whose label starts with char from active node
            if edge is none:
                insert a edge whose label starts with char
                insert index to the leaf node of the edge

                update suffix link
            else:
                update active point
                break

        else:
            if active_edge[active_length] == char:
                update active point

                if char is the last char:
                    add index to all the remainder suffix
                break
            else:
                split current active_edge at active_length with an internal node
                insert an new edge starts with char at the internal node
                insert index to the leaf node of the new edge

                update suffix link

        update active point

suffix links

在原来的后缀树中，只有中间结点会有suffix link，在本版本中，同一结点在表示a字符串的后缀树时是叶结点，则表示b字符串时可能就是中间结点。 所以我们统一给所以结点添加suffix link(根结点除外)。

添加上一个字符串所创建的suffix link，在添加下一个字符串时依然有效。
