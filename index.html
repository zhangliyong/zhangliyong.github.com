Suffix Tree

Fri 05 July 2013
By zhang liyong

In algorithm.

后缀树可以用来快速的对字符串进行搜索，详细信息推荐阅读：http://en.wikipedia.org/wiki/Suffix_tree，

后缀树的搜索非常快，但是构造后缀树比较复杂， Ukkonen提出的构造后缀树的算法时间复杂度为O(nlogn), 构造原理推荐阅读 http://stackoverflow.com/questions/9452701/ukkonens-suffix-tree-algorithm-in-plain-english

在继续阅读前确保你已经理解后缀树的构造原理。

上面构造的后缀树只表示一个字符串， 在很多情况下我们需要为多个字符串构造后缀树，例如查询包含'abc'的所有字符串， 当需要为n个字符串（如s1, s2, s3, ..., sn）构造时，目前通用的做法是找到n个不同的字符, u1, u2,..., un， 这n个字符不在这n个字符串中，然后组成一个字符串：s=s1u1s2u2.....snun，然后为s构造后缀树。

当有少量字符串的时候，这种方式比较实用，我们可以很轻易找到n个不同的字符，但当有大量字符串时， 我们很难找到这么多不同的字符，目前unicode的码点是从0到0x10ffff，也就是unicode最多能表示1114111个字符，仅仅是百万级。

所以通过不同字符来连接字符串的方案对大量字符串并不实用。

为此我们提供了另一种解决方案。 此种方案不需要n个不同的字符，我们需要为每个字符串设置一个惟一的索引，用于标识此字符串，即建立起索引到字符串的映射。

当时行搜索时，从根结点向下查找，当成功到达某一结点时，此结点及其所有子结点中的索引值即是查找结果。

例如查找'b'时，可找到索引1，2.

下面介绍构造此generalize suffix tree的过程。

给定字符串"aba"，其索引为0

  2. 将索引加入后缀树中的所有叶结点。
    • 如果最后一个字符加入后，remainder为0，则不需要做任何处理，此时后缀树已经创建成功；
  4. 如果remainder不为0, 则还有remainder个后缀没有对应到索引，根据需要创建中间结点，并将索引加到中间结点中。

suffix links

we are all talking about implicit node

if a suffix split an edge, then the smaller suffix must split an edge, so we can create a suffix link.

Prove

If the smaller suffix's next char is the test char, then the char is also the suffix's next char, because the test char is add before for all suffix.

所以有非叶结点非根结点都会有suffix link

证明不需要split edge中的suffix link那条语句

对于每一个字符串最后一个字符加入后，如果remainder不为0，说明还有多个suffix并没有添加index， 如果某一suffix结束于explicit结点，我们只需要将index加入到此结点即可， 测试用例：ababa 否则我们要对这些suffix创建explicit的结点，并加入index, 创建的结点之间也要建立suffix link， 测试用例：aa, aba

几个说明事项：

  1. 当为某一字符串构造了后缀树后，后缀树中会有suffix link, 当向这个后缀树中加入其它字符串后，这些suffix link依赖有效。
